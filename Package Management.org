1.Check if any helm charts installed
    helm ls

2.Search for a chart i want  
    helm search wordpress
3.Install stable wordpress
    helm install stable/wordpress
4.Upgrade example
    helm install stable/wordpress --version 5.0.1
     helm upgrade <helm-name> stable/wordpress
5.Download a chart 
    helm fetch stable/wordpress
6.Download a chart & untar
    helm fetch --untar stable/wordpress
7.Requirment missing example 
    rm -rf ./wordpress/charts/mariadb/
    helm dep list 
8.Download the missing requirments
    helm dep update 
9.Local install
    cd wordpress
    helm install .
10.Check mariadb excisting 
    kubectl get pods




The Helm Client
- Interacts with Tiller server
- Manages charts
- Creates new charts
- Packages charts into archives
- Interacts with chart repositories
- Installs and uninstalls charts
- Manages char release cycle

The Tiller server
- Interacts with the Kubernetes API
- Manages releases
- Listens for request from the helm client 
- Combines a chart and a config into a release
- Installs charts and tracks the release
- Upgrades and uninstalls charts

11.Check tiller pod 
    kubectl get pods --all-namespaces | grep tiller
